Manual Técnico — TourneyJS

Este documento describe la arquitectura, el analizador léxico (AFD), los módulos de cálculo de reportes y los artefactos generados.

Arquitectura

  • UI: index.html + style.css
  • Léxico: clase Lexer (AFD manual, sin regex)
  • Parser básico: clase Parser (consumo de tokens para poblar el modelo)
  • Reportes: funciones para estadísticas, goleadores e info general; generación de DOT

AFD (grafo)

Se incluye assets/afd.dot con el autómata de tokenización (símbolos, palabras, números y cadenas).

digraph AFD {
  rankdir=LR;
  node [shape=circle];
  INI -> WS    [label="espacios"];
  INI -> SYM   [label="{ } [ ] : ,"];
  INI -> CAD   [label="\""];
  INI -> NUM   [label="0-9"];
  INI -> ID    [label="letra/_"];
  ID -> ID     [label="letra/_/0-9"];
  NUM -> NUM   [label="0-9"];
  CAD -> CAD   [label="cualquiera/escape"];
  CAD -> ACEPT [label="\""];
}

Estados equivalentes y minimización

El diseño ya es compacto; para la práctica, la minimización no reduce estados adicionales por la separación de clases de equivalencia (letra/dígito/símbolo/espacio).

Restricciones cumplidas

  • Lenguaje: JavaScript
  • Aplicación web
  • AFD programado manualmente, procesando carácter por carácter
  • Sin uso de RegExp ni librerías de análisis (Jison, ANTLR, etc.)

Algoritmos y cálculos

  • Tokens: símbolo, cadena, número, reservada, identificador
  • Errores: caracteres no reconocidos, cadenas sin cerrar
  • Estadísticas: por equipo (J,G,P,GF,GC,GD), goleadores, general
  • Bracket: DOT conectando ganadores por orden (1-2 a semifinal 1, etc.)
